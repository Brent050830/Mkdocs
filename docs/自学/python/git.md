## git fetch、commit、push、pull、merge 详解

### 一张图理解五个命令

```python
远端仓库 (GitHub/Gitee)                本地仓库                    工作区
    ☁️  origin                          💻  .git                   📁 你的文件
                                                          
    ┌──────────┐      fetch        ┌──────────┐       add        ┌──────────┐
    │ 远端提交  │ ──────────────→  │ 本地提交  │ ←─────────────── │ 修改的文件 │
    │   A-B-C  │                  │   A-B-C  │                  │  hello.py │
    └──────────┘                  └──────────┘                  └──────────┘
         ↑                            │                              
         │          push              │                              
         └───────────────────────────┘                              
              ←  pull (fetch + merge) →
```

### commit — 提交到本地仓库

```python
工作区  ──git add──→  暂存区  ──git commit──→  本地仓库
```

把你修改过的文件**保存为一个版本快照**，记录到本地 `.git` 里。每次 commit 会生成一个唯一的哈希 ID（比如 `3559a8a`），就像游戏里的 " 存档点 "，以后可以随时回滚到这里。

### push — 推送到远端

```python
本地仓库  ──git push──→  远端仓库 (origin)
```

把你本地已经 commit 的提交**上传到 GitHub/Gitee 等远端**，别人就能看到你的代码了。

> 💡 **必须先在本地 commit，才能 push**

### fetch — 拉取远端信息（不合并）

```python
远端仓库  ──git fetch──→  本地仓库（仅更新远端分支的"镜像"）
```

把远端仓库的最新情况**下载到本地**，但**不会动你的代码**。你可以先看看别人更新了什么，再决定要不要合并。

> `fetch` = " 先看看有什么更新，不急着合并 "

### merge — 合并分支

```python
分支 A ──────────→
         ↘        ↘
          合并 → 新的提交（融合两边的修改）
         ↗        ↗
分支 B ──────────→
```

把一个分支的修改**合并到另一个分支**。比如你在 `feature` 分支开发了一个新功能，merge 到 `main` 分支就上线了。

### pull — 拉取并合并（一步到位）

```python
git pull = git fetch + git merge
```

从远端**下载更新并自动合并**到你的本地分支。最常用的同步命令。

> ⚠️ 如果你的本地有未推送的修改，pull 时可能产生**冲突**，需要手动解决。

### 一句话总结

| 命令       | 一句话         | 方向                 |
| -------- | ----------- | ------------------ |
| `commit` | 本地存档        | 工作区 → 本地（将修改的部分打包） |
| `push`   | 上传到远端       | 本地 → 远端（将打包的部分上传）  |
| `fetch`  | 查看远端更新（不合并） | 远端 → 本地（只下载）       |
| `merge`  | 合并两个分支      | 分支 → 分支（PR 相当于）    |
| `pull`   | 下载并合并       | 远端 → 本地（下载 + 合并）   |

```text
commit = 本地保存一个版本
push = 把本地版本上传到 GitHub
pull = 把 GitHub 上的新版本拉到本地
fetch = 只检查 GitHub 有没有新版本
```

> [!tip]+ 说明  
> 实际上就是云端和本地是一对一的（云端的一个分支对应本地的一个分支）  
> 合并是合并修改而不是覆盖
> 每个分支的云端和本地是相对应的
