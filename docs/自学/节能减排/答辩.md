## 一、车辆纵向动力学模型

### 1.1 物理参数

| 符号 | 含义 | 取值 |
|------|------|------|
| $m$ | 整车质量 | 1800 kg |
| $r_w$ | 车轮半径 | 0.32 m |
| $A_f$ | 迎风面积 | 2.4 m² |
| $C_d$ | 风阻系数 | 0.30 |
| $C_r$ | 滚阻系数 | 0.012 |
| $\rho$ | 空气密度 | 1.225 kg/m³ |
| $g$ | 重力加速度 | 9.81 m/s² |
| $i_g$ | 减速比 | 9.0 |
| $T_{\max}$ | 电机最大扭矩 | 320 Nm |
| $n_{\max}$ | 电机最大转速 | 13000 rpm |
| $n_{\text{base}}$ | 基速 | 4000 rpm |
| $P_{\max}$ | 电机最大功率 | $T_{\max} \cdot n_{\text{base}} \cdot 2\pi / 60$ |
| $\eta_d$ | 传动效率 | 0.95 |
| $\delta$ | 旋转质量系数 | 1.05 |
| $\Delta s$ | 距离步长 | 1.0 m |
| $Q_{\text{bat}}$ | 电池容量 | 60000 Wh |

### 1.2 电机转速与车速的关系

$$n_k = \frac{v_k}{r_w} \cdot i_g \cdot \frac{60}{2\pi} \quad [\text{rpm}]$$

### 1.3 电机扭矩限幅

当 $n_k > n_{\text{base}}$ 时，受恒功率约束：

$$T_k^{\text{drive\_limit}} = \min\!\left(T_{\max},\; \frac{P_{\max}}{n_k \cdot 2\pi / 60}\right)$$

$$T_k^{\text{regen\_limit}} = \min\!\left(0.5 \, T_{\max},\; \frac{0.5 \, P_{\max}}{n_k \cdot 2\pi / 60}\right)$$

$$T_k = \text{clip}\!\left(T_k^{\text{cmd}},\; -T_k^{\text{regen\_limit}},\; T_k^{\text{drive\_limit}}\right)$$

### 1.4 轮端受力分析

$$F_{\text{roll}} = m g \, C_r \cos\theta_k$$

$$F_{\text{grade}} = m g \sin\theta_k$$

$$F_{\text{air}} = \tfrac{1}{2} \rho \, C_d \, A_f \, v_k^2$$

$$F_{\text{drive}} = \frac{T_k \cdot i_g \cdot \eta_d^*}{r_w}, \quad \eta_d^* = \begin{cases} \eta_d & T_k \geq 0 \\ 1/\eta_d & T_k < 0 \end{cases}$$

$$F_{\text{net}} = F_{\text{drive}} - F_{\text{roll}} - F_{\text{grade}} - F_{\text{air}}$$

### 1.5 状态转移（基于距离步进）

$$a_k = \frac{F_{\text{net}}}{\delta \, m}$$

$$v_{k+1} = \sqrt{\max\!\left(v_k^2 + 2\,a_k\,\Delta s,\;\; 0.25\right)}$$

$$\bar{v}_k = \max\!\left(\frac{v_k + v_{k+1}}{2},\;\; 0.5\right)$$

$$\Delta t_k = \frac{\Delta s}{\bar{v}_k}$$

### 1.6 电机效率查询

$$\eta_k = \mathcal{M}(T_k,\, \bar{n}_k) \quad \in [0.01,\, 0.99]$$

其中 $\mathcal{M}$ 为二维双线性插值效率表（来自 `sys_eff_pivot.csv`），$\bar{n}_k$ 为基于区间均速的电机转速。

### 1.7 电池功率与能量

$$P_{\text{motor}} = T_k \cdot \bar{n}_k \cdot \frac{2\pi}{60}$$

$$P_{\text{bat}} = \begin{cases} P_{\text{motor}} / \eta_k & P_{\text{motor}} \geq 0 \quad \text{(驱动)} \\ P_{\text{motor}} \cdot \eta_k & P_{\text{motor}} < 0 \quad \text{(回收)} \end{cases}$$

$$E_k^{\text{bat}} = \frac{P_{\text{bat}} \cdot \Delta t_k}{3600} \quad [\text{Wh}]$$

$$\text{SOC}_{k+1} = \text{SOC}_k - \frac{E_k^{\text{bat}}}{Q_{\text{bat}}}$$

### 1.8 损耗能量（用于奖励计算的口径）

$$F_w = \delta m a_k + m g C_r \cos\theta_k + m g \sin\theta_k + \tfrac{1}{2}\rho C_d A_f \bar{v}_k^2$$

$$\lambda_{\text{loss}} = \begin{cases} \max(0,\; 1/\eta_k - 1) & F_w \geq 0 \\ \max(0,\; 1 - \eta_k) & F_w < 0 \end{cases}$$

$$E_k^{\text{loss}} = \frac{|F_w| \cdot \lambda_{\text{loss}} \cdot \Delta s}{3600} \quad [\text{Wh}]$$

## 二、动作空间（Action Space）

$$\boxed{a_k \in [-1,\, 1] \quad \text{（1维连续动作）}}$$

Agent 输出的动作 $a_k$ 被映射为**残差扭矩**：

$$\Delta T_k = a_k \times \sigma_T$$

其中残差扭矩缩放系数 $\sigma_T = 10.0$ Nm（可配置）。

**最终发送给电机的指令扭矩**：

$$T_k^{\text{cmd}} = T_k^{\text{ref}} + \Delta T_k$$

即在参考轨迹的扭矩 $T_k^{\text{ref}}$ 基础上叠加一个 $[-\sigma_T, +\sigma_T]$ 范围的微调量。

## 四、奖励函数（Reward Function）

### 4.1 训练分两阶段

| 阶段 | mode | 目的 |
|------|------|------|
| 第一阶段 | `track` | 纯跟踪（不考虑节能） |
| 第二阶段 | `energy` | 在跟踪基础上追求节能 |

### 4.2 跟踪惩罚项（两阶段共用）

$$\mathcal{P}_{\text{track}} = 8 \left(\frac{|v_k - v_k^{\text{ref}}|}{\epsilon_v}\right)^2 + 4 \left(\frac{|d_k - d_k^{\text{ref}}|}{\epsilon_d}\right)^2 + 2 \left(\frac{|a_k - a_k^{\text{ref}}|}{\epsilon_a}\right)^2$$

各容差 $\epsilon$ 按风格不同：

| 风格 | $\epsilon_v$ (m/s) | $\epsilon_d$ (m) | $\epsilon_a$ (m/s²) |
|------|-----|-----|------|
| eco | 0.20 | 0.30 | 0.25 |
| normal | 0.25 | 0.40 | 0.35 |
| sport | 0.30 | 0.50 | 0.50 |

---

### 4.3 第一阶段：Track 模式

$$\boxed{r_k^{\text{track}} = -\mathcal{P}_{\text{track}} - 0.1 \left(\frac{|T_k - T_{k-1}|}{8.0}\right)^2}$$

纯跟踪 + 平顺性惩罚。不涉及能量项。

---

### 4.4 第二阶段：Energy 模式

**Step 1 — 计算单步能量偏差密度**

$$\Delta L_k = \frac{E_k^{\text{agent,loss}}}{\Delta s} - \frac{E_k^{\text{ref,loss}}}{\Delta s}$$

> $\Delta L_k > 0$：Agent 比参考消耗更多损耗能量（惩罚）  
> $\Delta L_k < 0$：Agent 比参考更节能（奖励）

**Step 2 — 能量奖励项**

$$r_{\text{energy}} = -w_c \cdot \Delta L_k - w_u \left(\frac{|T_k - T_{k-1}|}{\sigma_T}\right)^2$$

其中：
- $w_c = w_{\text{base}} \times s_{\text{style}} \times 200$，这里 $w_{\text{base}}$ 为当前能量权重（从 0 线性增长到 $w_{\text{final}}$），$s_{\text{style}}$ 为风格缩放系数（eco=1.0, normal=0.8, sport=0.5）
- $w_u = 0.08$，扭矩抖动惩罚权重

**Step 3 — 连续门控融合**

$$g = \exp\!\left(-\beta \cdot \mathcal{P}_{\text{track}}\right), \quad \beta = 0.05$$

$$\boxed{r_k^{\text{energy}} = -\mathcal{P}_{\text{track}} + g \cdot r_{\text{energy}}}$$

> [!IMPORTANT]  
> 门控函数 $g \in (0, 1]$ 的设计意图：
> - 当跟踪误差很小（$\mathcal{P}_{\text{track}} \to 0$）时，$g \to 1$，Agent 完全感受到能量奖励
> - 当跟踪误差较大时，$g \to 0$，能量奖励被 " 关闭 "，Agent 被迫先修复跟踪
> - 这避免了 Agent 通过 " 牺牲跟踪来换取节能 " 的作弊策略

---

### 4.5 终止奖励（Episode 结束时一次性给予）

**情况 A — 跟踪合格 `tracking_ok = True`**

$$r_{\text{terminal}} = 10 \cdot s_{\text{style}} \cdot \underbrace{\frac{E_{\text{ref}} - E_{\text{agent}}}{E_{\text{ref}} + \varepsilon}}_{\text{总节能率}} - w_t \cdot \underbrace{\left(\frac{t_{\text{agent}} - t_{\text{ref}}}{t_{\text{ref}} + \varepsilon}\right)^2}_{\text{时间偏差惩罚}}$$

其中 $w_t$ 为终止时间权重（eco=4.0, normal=5.0, sport=5.0）。

**情况 B — 跟踪不合格**

$$r_{\text{terminal}} = -20 \left[\left(\frac{\text{MAE}_v}{\epsilon_v}\right)^2 + \left(\frac{\text{MAE}_d}{\epsilon_d}\right)^2\right]$$

## 两套能量口径详解

系统中存在**两套不同的能量计算**，分别用于不同的奖励环节：

| | 口径 A：损耗能量 $E^{\text{loss}}$ | 口径 B：电池能量 $E^{\text{bat}}$ |
|---|---|---|
| **用在哪** | 每步奖励（energy 模式） | 终局奖励 |
| **代码变量** | `cmp_energy_step` | `energy_step` |
| **代码函数** | `compute_loss_energy()` | `calc_battery_power_w()` × Δt |
| **物理含义** | 电机内部因效率不足产生的**热损耗** | 电池端实际**输出/吸收的总能量** |
| **累加器** | `total_cmp_energy_agent` | `total_energy_agent` |

### 4.6 硬约束违反惩罚

每步检查：

$$\text{if} \quad |v_k - v_k^{\text{ref}}| > 2\epsilon_v \;\;\text{or}\;\; |d_k - d_k^{\text{ref}}| > 2\epsilon_d \quad \Rightarrow \quad r_k \mathrel{-}= 50$$

### 口径 A：损耗能量 $E_k^{\text{loss}}$（每步奖励用）

> 代码位置：[compute_loss_energy()](file:///c:/Users/17871/Desktop/节能/motor_env.py#L216-L234)

#### 计算链路

**Step 1 — 计算轮端合力**

$$F_w = \underbrace{\delta m a_k}_{\text{加速力}} + \underbrace{m g C_r \cos\theta_k}_{\text{滚阻}} + \underbrace{m g \sin\theta_k}_{\text{坡度阻力}} + \underbrace{\tfrac{1}{2}\rho C_d A_f \bar{v}_k^2}_{\text{空气阻力}}$$

> 注意：这里用的是**均速** $\bar{v}_k$，**当步加速度** $a_k$（与当前状态转移一致）

**Step 2 — 计算损失系数**

$$\lambda_{\text{loss}} = \begin{cases} \max\!\left(0,\;\; \dfrac{1}{\eta_k} - 1\right) & \text{if } F_w \geq 0 \quad \text{（驱动工况）} \\[8pt] \max\!\left(0,\;\; 1 - \eta_k\right) & \text{if } F_w < 0 \quad \text{（制动回收工况）} \end{cases}$$

> **物理解释**：
> - 驱动时：电机效率 $\eta=0.90$ → 损失系数 $= 1/0.90 - 1 = 0.111$，即每 1J 有用功需额外消耗 0.111J 的热量
> - 回收时：电机效率 $\eta=0.90$ → 损失系数 $= 1 - 0.90 = 0.10$，即每 1J 的动能只有 0.10J 被浪费为热

**Step 3 — 计算损耗能量**

$$\boxed{E_k^{\text{loss}} = \frac{|F_w| \cdot \lambda_{\text{loss}} \cdot \Delta s}{3600} \quad [\text{Wh}]}$$

#### 在奖励中的使用

```python
# motor_env.py L783-L788
ref_loss_density  = ref_cmp_energy_k / ds     # 参考轨迹的损耗密度
agent_loss_density = cmp_energy_step / ds      # Agent的损耗密度
loss_delta = agent_loss_density - ref_loss_density  # 正 = 多耗损, 负 = 更高效

r_energy = -w_c * loss_delta - torque_penalty
```

$$\Delta L_k = \frac{E_{k,\text{agent}}^{\text{loss}}}{\Delta s} - \frac{E_{k,\text{ref}}^{\text{loss}}}{\Delta s}$$

$$r_{\text{energy}} = -w_c \cdot \Delta L_k - w_u\!\left(\frac{|T_k - T_{k-1}|}{\sigma_T}\right)^2$$

> [!IMPORTANT]  
> **口径 A 衡量的是 " 电机把多少有用功变成了废热 "**。  
> Agent 降低 $E^{\text{loss}}$ 的途径有两条：
> 1. 让电机工作在效率更高的区域（$\eta \uparrow$ → $\lambda_{\text{loss}} \downarrow$）
> 2. 降低轮端受力 $|F_w|$（即减速/减扭矩 — 这是 " 作弊途径 "）

### 口径 B：电池能量 $E_k^{\text{bat}}$（终局奖励用）

> 代码位置：[calc_battery_power_w()](file:///c:/Users/17871/Desktop/节能/motor_env.py#L185-L195) + [step_longitudinal_dynamics() L277](file:///c:/Users/17871/Desktop/节能/motor_env.py#L277)

#### 计算链路

**Step 1 — 计算电机机械功率**

$$P_{\text{motor}} = T_k \cdot \bar{n}_k \cdot \frac{2\pi}{60} \quad [\text{W}]$$

**Step 2 — 经效率折算为电池功率**

$$P_{\text{bat}} = \begin{cases} \dfrac{P_{\text{motor}}}{\eta_k} & \text{if } P_{\text{motor}} \geq 0 \quad \text{（驱动：电池输出 > 机械功）} \\[8pt] P_{\text{motor}} \cdot \eta_k & \text{if } P_{\text{motor}} < 0 \quad \text{（回收：电池吸收 < 机械功）} \end{cases}$$

**Step 3 — 计算电池能量**

$$\boxed{E_k^{\text{bat}} = \frac{P_{\text{bat}} \cdot \Delta t_k}{3600} \quad [\text{Wh}]}$$

其中 $\Delta t_k = \Delta s / \bar{v}_k$。

**Step 4 — 全程累加得到总电池消耗**

$$E_{\text{total}}^{\text{bat}} = \sum_{k=0}^{N-1} E_k^{\text{bat}}$$

代码中：`self.total_energy_agent += energy_step`（[L605](file:///c:/Users/17871/Desktop/节能/motor_env.py#L605)）

#### 在终局奖励中的使用

```python
# motor_env.py L818-L821
ref_total   = ref["ref_total_energy"]         # 参考轨迹的 Σ E_bat
agent_total = self.total_energy_agent          # Agent 的 Σ E_bat
reward_saving = (ref_epd - agent_epd) / (ref_epd + ε)
```

$$\text{saving\_ratio} = \frac{E_{\text{ref}}^{\text{bat}} / D - E_{\text{agent}}^{\text{bat}} / D}{E_{\text{ref}}^{\text{bat}} / D + \varepsilon}$$

$$\boxed{r_{\text{terminal}} = 10 \cdot s_{\text{style}} \cdot \text{saving\_ratio} - w_t \left(\frac{t_{\text{agent}} - t_{\text{ref}}}{t_{\text{ref}}}\right)^2}$$

> [!IMPORTANT]  
> **口径 B 衡量的是 " 电池实际放了多少电 "**。这是最终用户真正关心的指标。  
> 它同时包含了有用功 + 热损耗，即：$E^{\text{bat}} = E^{\text{useful}} + E^{\text{loss}}$

---

### 两者的关系

用一个驱动工况的例子说明（$F_w > 0$）：

$$E^{\text{bat}} = \frac{|F_w| \cdot \Delta s}{\eta_k \cdot 3600} = \frac{|F_w| \cdot \Delta s}{3600} + \frac{|F_w| \cdot (1/\eta_k - 1) \cdot \Delta s}{3600}$$

$$\boxed{E^{\text{bat}} = \underbrace{E^{\text{useful}}}_{\text{有用功（做功本身）}} + \underbrace{E^{\text{loss}}}_{\text{热损耗（口径A）}}}$$

即：
- **口径 A** ($E^{\text{loss}}$) 只关注**效率损失的部分**（热量浪费）
- **口径 B** ($E^{\text{bat}}$) 是电池端的**全部消耗**（有用功 + 热损耗）

---

### 数值算例

假设某一步：$T_k = 80$ Nm，$\bar{n}_k = 7000$ rpm，$\eta_k = 0.93$，$\Delta s = 1$ m，$\bar{v}_k = 25$ m/s

**口径 B（电池能量）**：  
$$P_{\text{motor}} = 80 \times 7000 \times \frac{2\pi}{60} = 58643 \text{ W}$$  
$$P_{\text{bat}} = 58643 / 0.93 = 63057 \text{ W}$$  
$$\Delta t = 1.0 / 25 = 0.04 \text{ s}$$  
$$E^{\text{bat}} = 63057 \times 0.04 / 3600 = 0.701 \text{ Wh}$$

**口径 A（损耗能量）**：  
假设 $F_w = 1200$ N（轮端合力）  
$$\lambda_{\text{loss}} = 1/0.93 - 1 = 0.0753$$  
$$E^{\text{loss}} = 1200 \times 0.0753 \times 1.0 / 3600 = 0.0251 \text{ Wh}$$

**验证**：  
$$E^{\text{useful}} = 1200 \times 1.0 / 3600 = 0.333 \text{ Wh}$$  
$$E^{\text{useful}} + E^{\text{loss}} = 0.333 + 0.0251 \approx 0.358 \text{ Wh}$$

> 注：此处 $E^{\text{bat}}$ 与 $E^{\text{useful}} + E^{\text{loss}}$ 的差异，来源于口径 A 的 $F_w$ 用的是力学公式估算（含加速力），  
> 而口径 B 的 $P_{\text{motor}}$ 用的是实际扭矩×转速，两者在物理上是近似一致的。

### 总结流程图

```python
电机实际工况 (T_k, n_k, η_k)
          │
          ├── 口径A路径 ────────────────────────────────────────┐
          │   F_w = δma + mgCr·cosθ + mgsinθ + ½ρCdA·v²       │
          │   λ_loss = 1/η - 1                                 │
          │   E_loss = |F_w| · λ_loss · Δs / 3600              │
          │                                                     │
          │   → 每步奖励：ΔL = E_loss_agent/Δs - E_loss_ref/Δs │
          │   → r_energy = -w_c · ΔL                           │
          │                                                     │
          ├── 口径B路径 ────────────────────────────────────────┐
          │   P_motor = T · n · 2π/60                           │
          │   P_bat = P_motor / η                               │
          │   E_bat = P_bat · Δt / 3600                         │
          │                                                     │
          │   → 全程累加：E_total = Σ E_bat                     │
          │   → 终局奖励：saving = (E_ref - E_agent) / E_ref   │
          │   → r_terminal = 10 · s_style · saving              │
          └─────────────────────────────────────────────────────┘
```

## 五、约束系统（Lagrangian 乘子）

环境跟踪五类约束的违反量：

| 约束名称 | 计算公式 |
|---------|---------|
| 速度 | $c_v = \max(0,\; \|v - v^{\text{ref}}\| / \epsilon_v - 1)$ |
| 距离 | $c_d = \max(0,\; \|d - d^{\text{ref}}\| / \epsilon_d - 1)$ |
| 平顺性 | $c_s = \max(0,\; \|T_k - T_{k-1}\| / 8.0 - 1)$ |
| 投影 | $c_p = \frac{1}{H}\sum_{h=1}^{H}\max\!\left(0,\; \frac{\|v_k + a_k \cdot h \Delta t - v_{k+h}^{\text{ref}}\|}{\epsilon_v} - 1.5\right)$ |
| 窗口能量 | $c_w = \max\!\left(0,\; \frac{\sum_{i}^{W} E_i^{\text{agent,loss}}}{\sum_{i}^{W} E_i^{\text{ref,loss}} + \varepsilon} - 1.3\right)$ |

外层训练使用 Lagrangian 乘子 $\lambda_j$ 对每个约束加权，在 episode 间自适应更新。

## 六、网络架构

$$\pi_\phi(a_k \mid \mathbf{o}_k): \quad \mathbb{R}^{28} \xrightarrow{\text{FC}(128)} \text{ReLU} \xrightarrow{\text{FC}(128)} \text{ReLU} \xrightarrow{\text{FC}(1)} \tanh$$

$$V_\psi(\mathbf{o}_k): \quad \mathbb{R}^{28} \xrightarrow{\text{FC}(128)} \text{ReLU} \xrightarrow{\text{FC}(128)} \text{ReLU} \xrightarrow{\text{FC}(1)} \mathbb{R}$$
